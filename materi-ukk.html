2️⃣ Koneksi Database (PDO) - Detail

PDO = PHP Data Objects. Cara modern dan aman untuk mengakses database dari PHP.

📄 app/config/koneksi.php
koneksi.php - KODE LENGKAP PHP 
<?php
// Konfigurasi database
$host = 'localhost';              // Baris 8
$dbname = 'inventaris_fakhri';    // Baris 9
$username = 'root';               // Baris 10
$password = '';                   // Baris 11

try {
    // Buat koneksi PDO
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $username,
        $password,
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES => false
        ]
    );
} catch (PDOException $e) {
    die("Koneksi database gagal: " . $e->getMessage());
}
?>

Penjelasan Per Baris

8-11

Variabel konfigurasi: $host = server database, $dbname = nama database, $username & $password = kredensial MySQL

15-16

new PDO(...): Membuat objek koneksi baru. Format: "mysql:host=...;dbname=..."

21

ERRMODE_EXCEPTION: Kalau ada error, PHP akan throw exception (bisa ditangkap dengan try-catch)

24

FETCH_ASSOC: Hasil query jadi array dengan key = nama kolom. Contoh: $row['nama_barang']

27

EMULATE_PREPARES = false: Pakai prepared statement asli dari MySQL, bukan emulasi PHP (lebih aman)

30-31

catch (PDOException): Kalau koneksi gagal, tampilkan pesan error dan hentikan script

✅ Kenapa Pakai PDO?

  • Keamanan: Prepared statement mencegah SQL Injection
  • Fleksibel: Bisa untuk MySQL, PostgreSQL, SQLite
  • Error Handling: Dengan try-catch lebih mudah debug

3️⃣ Session & Sistem Login - Detail

Session = cara menyimpan data user sementara di server selama browsing.

🔄 Alur Login Lengkap
1 User buka login.php

Cek apakah sudah login (session ada?)

2 User isi form & submit

Data dikirim via POST

3 PHP query database

SELECT * FROM users WHERE username = ?

4 Verifikasi password

password_verify($input, $hash)

5 Simpan ke session

$_SESSION['user_id'] = $user['id']

6 Redirect ke dashboard

header('Location: dashboard.php')

📄 app/auth/login.php
login.php - KODE PENTING PHP 
session_start();  // Baris 7 - WAJIB di awal!

// Cek sudah login? Redirect ke dashboard
if (isset($_SESSION['user_id'])) {
    header('Location: dashboard.php');
    exit;
}

// Proses form login
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = trim($_POST['username'] ?? '');
    $password = $_POST['password'] ?? '';
    
    // Query ke database dengan prepared statement
    $stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
    $stmt->execute([$username]);
    $user = $stmt->fetch();
    
    // Verifikasi password
    if ($user && password_verify($password, $user['password'])) {
        // LOGIN BERHASIL!
        $_SESSION['user_id'] = $user['id'];
        $_SESSION['username'] = $user['username'];
        
        header('Location: dashboard.php');
        exit;
    } else {
        $error = 'Username atau password salah!';
    }
}

Penjelasan Fungsi Penting

Fungsi Kegunaan
session_start() Memulai session. WAJIB di awal file sebelum output HTML!
$_SESSION['key'] Superglobal untuk menyimpan/membaca data session
isset($_SESSION['key']) Cek apakah session dengan key tersebut ada
password_verify($input, $hash) Membandingkan password plain dengan hash di database
header('Location: ...') Redirect ke halaman lain
exit Hentikan script setelah redirect

⚠️ Aturan Penting Session

session_start() HARUS dipanggil sebelum ada output HTML apapun (termasuk spasi kosong di awal file!)

CRUD = Create, Read, Update, Delete. 4 operasi dasar untuk mengelola data.

Read SELECT app/barang/index.php
Update UPDATE ... SET app/barang/edit.php

CREATE - Tambah Data

📄app/barang/tambah.php
INSERT dengan Prepared StatementPHP
// Ambil data dari form
$nama_barang = trim($_POST['nama_barang'] ?? '');
$stok = (int) ($_POST['stok'] ?? 0);
$harga = (float) ($_POST['harga'] ?? 0);

// Prepared statement - AMAN dari SQL Injection!
$stmt = $pdo->prepare("INSERT INTO barang (nama_barang, stok, harga) VALUES (?, ?, ?)");
$stmt->execute([$nama_barang, $stok, $harga]);

// Penjelasan:
// prepare() = siapkan query dengan placeholder ?
// execute([...]) = jalankan query, isi placeholder dengan nilai dari array

UPDATE - Edit Data

📄app/barang/edit.php
UPDATE dengan WHEREPHP 
// UPDATE - HARUS pakai WHERE untuk menentukan baris mana!
$stmt = $pdo->prepare("UPDATE barang SET nama_barang = ?, stok = ?, harga = ? WHERE id = ?");
$stmt->execute([$nama_barang, $stok, $harga, $id]);

// PENTING: Tanpa WHERE, SEMUA baris akan terupdate!

DELETE - Hapus Data

📄app/barang/hapus.php
DELETE dengan WHEREPHP 
// DELETE - WAJIB pakai WHERE!

$stmt->execute([$id]);

❌ Kesalahan Fatal

UPDATE atau DELETE tanpa WHERE akan mempengaruhi SEMUA baris di tabel!

7️⃣ Database Transaction

Transaction memastikan beberapa query berjalan SEMUA berhasil atau SEMUA dibatalkan.

🔄 Alur Transaction
1 beginTransaction()

Mulai mode transaction

2 Query 1: INSERT transaksi

Belum permanen

3 Query 2: UPDATE stok

Belum permanen

4 commit() atau rollback()

Simpan semua ATAU batalkan semua

📄app/transaksi/masuk.php
Transaction untuk Barang MasukPHP
try {
    $pdo->beginTransaction();  // Mulai transaction
    
    // Query 1: Insert ke tabel transaksi
    $stmt = $pdo->prepare("INSERT INTO transaksi (...) VALUES (...)");
    $stmt->execute([...]);
    
    // Query 2: Update stok barang (TAMBAH)
    $stmt = $pdo->prepare("UPDATE barang SET stok = stok + ? WHERE id = ?");
    $stmt->execute([$jumlah, $id_barang]);
    
    $pdo->commit();  // SIMPAN semua jika berhasil
    
} catch (Exception $e) {
    $pdo->rollback();  // BATALKAN semua jika ada error
    $error = 'Terjadi kesalahan!';
}

💡 Kenapa Pakai Transaction?

Bayangkan INSERT berhasil tapi UPDATE gagal. Stok jadi tidak sinkron! Dengan transaction, kalau ada yang gagal, semuanya dibatalkan.
