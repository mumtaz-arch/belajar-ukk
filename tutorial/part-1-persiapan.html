Apa itu Inventaris?

Inventaris adalah catatan barang-barang yang dimiliki. Contoh: toko mencatat berapa banyak laptop, mouse, keyboard yang ada di gudang. Kalau ada barang masuk (beli dari supplier), stok bertambah. Kalau ada barang keluar (dijual), stok berkurang.

1 Struktur Folder Project

Pertama, kita buat folder-folder untuk menyimpan file PHP. Ini penting supaya project kita rapi dan mudah dicari.

bahan-ajar-ukk/ ← Folder utama project ├── database.sql ← File untuk membuat database ├── index.php ← Halaman pertama (redirect ke login) ├── app/ ← Folder aplikasi utama │ ├── config/ ← Pengaturan │ │ └── koneksi.php ← Koneksi ke database │ ├── includes/ ← Template yang dipakai berulang │ │ ├── header.php ← Bagian atas halaman (menu) │ │ └── footer.php ← Bagian bawah halaman │ ├── auth/ ← Login dan logout │ │ ├── login.php │ │ └── logout.php │ ├── barang/ ← CRUD data barang │ │ ├── index.php ← Lihat semua barang │ │ ├── tambah.php ← Tambah barang baru │ │ ├── edit.php ← Edit barang │ │ └── hapus.php ← Hapus barang │ ├── transaksi/ ← Barang masuk/keluar │ └── dashboard.php ← Halaman utama setelah login

Kenapa Harus Dipisah?

  • Mudah dicari - Kalau mau edit barang, langsung buka folder barang/
  • Rapi - Tidak campur aduk semua file di satu tempat
  • Profesional - Ini cara kerja programmer sungguhan

2 Membuat Database

Database adalah tempat menyimpan data. Bayangkan seperti lemari arsip digital. Kita butuh 3 tabel:

  • users - Menyimpan data user yang bisa login
  • barang - Menyimpan data barang (nama, stok, harga)
  • transaksi - Menyimpan catatan barang masuk/keluar

Langkah 1: Buka phpMyAdmin

Buka browser, ketik: localhost/phpmyadmin

Langkah 2: Copy-paste kode SQL ini

database.sql SQL 
-- Buat database baru
CREATE DATABASE IF NOT EXISTS inventaris_fakhri;
USE inventaris_fakhri;

-- Tabel users (untuk login)
CREATE TABLE IF NOT EXISTS users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- Tabel barang
CREATE TABLE IF NOT EXISTS barang (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama_barang VARCHAR(100) NOT NULL,
    stok INT DEFAULT 0,
    harga DECIMAL(12,2) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- Tabel transaksi
CREATE TABLE IF NOT EXISTS transaksi (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_barang INT NOT NULL,
    jenis_transaksi ENUM('masuk', 'keluar') NOT NULL,
    jumlah INT NOT NULL,
    tanggal_transaksi DATETIME DEFAULT CURRENT_TIMESTAMP,
    keterangan TEXT,
    FOREIGN KEY (id_barang) REFERENCES barang(id) ON DELETE CASCADE
) ENGINE=InnoDB;

Penjelasan Kode SQL

CREATE DATABASE = Membuat database baru bernama "inventaris_fakhri"
CREATE TABLE = Membuat tabel baru di dalam database
INT = Tipe data angka bulat (1, 2, 3, dst)
VARCHAR(100) = Tipe data teks maksimal 100 karakter
AUTO_INCREMENT = Angka otomatis bertambah (1, 2, 3...)
PRIMARY KEY = Kolom utama yang unik, tidak boleh sama
FOREIGN KEY = Menghubungkan tabel transaksi ke tabel barang

3 File Koneksi Database

Sekarang kita buat file PHP untuk menghubungkan aplikasi ke database. File ini akan dipakai di semua halaman.

app/config/koneksi.php PHP 
<?php
/**
 * File Koneksi Database
 * Menggunakan PDO untuk keamanan
 */

// Pengaturan database
$host = 'localhost';           // Alamat server database
$dbname = 'inventaris_fakhri'; // Nama database
$username = 'root';            // Username MySQL (default: root)
$password = '';                // Password MySQL (default: kosong)

try {
    // Membuat koneksi ke database
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $username,
        $password,
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES => false
        ]
    );
} catch (PDOException $e) {
    // Kalau gagal koneksi, tampilkan pesan error
    die("Koneksi gagal: " . $e->getMessage());
}

Penjelasan Baris per Baris

<?php = Tanda mulai kode PHP
$host = 'localhost' = Variabel untuk menyimpan alamat server. Localhost artinya di komputer sendiri.
$dbname = 'inventaris_fakhri' = Nama database yang sudah kita buat tadi
new PDO(...) = Membuat objek koneksi ke database. PDO adalah cara aman untuk koneksi database di PHP.
try { } catch { } = Cara menangani error. Kalau ada error di dalam try, akan ditangkap di catch.
die("...") = Menghentikan program dan menampilkan pesan

Kenapa PDO?

PDO lebih aman dari mysql_query yang sudah kuno. PDO bisa mencegah serangan SQL Injection - yaitu hacker yang coba merusak database lewat form input.

4 Cara Menggunakan Koneksi

Setelah file koneksi dibuat, kita bisa pakai di file PHP lain. Berikut contoh penggunaannya:

Contoh penggunaan PHP 
<?php
// 1. Panggil file koneksi
require_once __DIR__ . '/../config/koneksi.php';

// 2. Ambil semua data barang
$stmt = $pdo->query("SELECT * FROM barang");
$data_barang = $stmt->fetchAll();

// 3. Tampilkan data
foreach ($data_barang as $barang) {
    echo $barang['nama_barang'] . "<br>";
}

Penjelasan

require_once = Memanggil file lain. "once" artinya hanya sekali (tidak duplikat)
__DIR__ = Lokasi folder file ini berada
$pdo->query("...") = Menjalankan perintah SQL ke database
fetchAll() = Mengambil semua hasil dalam bentuk array
foreach = Perulangan untuk mengakses setiap data satu per satu

Contoh INSERT (Menambah Data)

Menambah barang baru PHP 
<?php
require_once __DIR__ . '/../config/koneksi.php';

// Siapkan data
$nama = "Laptop ASUS";
$stok = 10;
$harga = 15000000;

// Masukkan ke database (cara aman dengan prepare)
$stmt = $pdo->prepare("INSERT INTO barang (nama_barang, stok, harga) VALUES (?, ?, ?)");
$stmt->execute([$nama, $stok, $harga]);

echo "Barang berhasil ditambahkan!";

Kenapa Pakai prepare() dan execute()?

Ini namanya Prepared Statement. Tanda ? adalah placeholder yang akan diisi oleh array di execute(). Ini mencegah SQL Injection karena data user tidak langsung dimasukkan ke query.
