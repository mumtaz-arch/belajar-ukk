1️⃣ Pendahuluan

Apa itu Version Control?

Version Control adalah sistem yang merekam perubahan file dari waktu ke waktu, sehingga kamu bisa melihat versi sebelumnya kapan saja.

💡 Analogi Sederhana

Bayangkan kamu main game. Setiap level, kamu SAVE progress-nya. Kalau mati, kamu bisa LOAD dari save point terakhir. Git itu kayak sistem SAVE/LOAD untuk kode program!

Masalah Tanpa Git

Pernahkah kamu mengalami ini?

  • ❌ "project_final.php" → "project_final_fix.php" → "project_final_fix_beneran.php" → "project_ASLI_JANGAN_DIUBAH.php"
  • ❌ Kerja kelompok: "Siapa yang ubah file ini?! Kok error?!"
  • ❌ "Kemarin kodenya jalan, sekarang error. Gimana balikkin?"
  • ❌ Flash disk hilang = project hilang 😭

✅ Dengan Git

  • Semua versi tersimpan rapi dengan keterangan
  • Bisa balik ke versi sebelumnya kapan saja
  • Kerja kelompok jadi gampang, tau siapa ubah apa
  • Project aman di cloud (GitHub)

2️⃣ Pengenalan Git

Apa itu Git?

Git adalah software version control yang paling populer di dunia. Dibuat oleh Linus Torvalds (yang juga buat Linux) tahun 2005.

Cara Kerja Git (Sederhana)

🔄 Alur Kerja Git
1. Edit File

Kamu ngoding seperti biasa

2. git add

Tandai file yang mau disimpan

3. git commit

Simpan snapshot dengan pesan

4. git push

Upload ke GitHub (opsional)

Game Git
Save slot Commit
Load game git checkout / git revert
New game+ Branch baru
Cloud save GitHub

Buka Git Bash atau Command Prompt, ketik:

Kalau muncul versi, berarti sukses! ✅

WAJIB dilakukan sebelum pakai Git:

4️⃣ Konsep Dasar Git

Repository (Repo)

Repository = folder project yang "dipantau" oleh Git. Di dalamnya ada folder tersembunyi .git yang menyimpan semua history.

Tiga Area Utama Git

📊 Alur Data di Git
Working Directory

Folder tempat kamu edit file

Staging Area

Ruang tunggu sebelum commit

Repository (.git)

Database history

Area Penjelasan Command
Working Directory Folder biasa tempat kamu kerja & edit file -
Staging Area Penampungan sementara file yang siap di-commit git add
Repository Database yang menyimpan semua commit/history git commit

Commit

Commit = snapshot/foto dari project pada waktu tertentu. Setiap commit punya:

  • 🔑 Hash ID - kode unik (contoh: a1b2c3d)
  • 📝 Message - keterangan perubahan
  • 👤 Author - siapa yang commit
  • 📅 Date - kapan commit dilakukan

Branch (Pengantar)

Branch = cabang development. Bayangkan seperti timeline alternatif. Kamu bisa bikin fitur baru di branch terpisah tanpa mengganggu kode utama.

5️⃣ Perintah Dasar Git

git init

Membuat folder jadi repository Git (inisialisasi).

Inisialisasi RepositoryGIT 
# Masuk ke folder project
cd project-website

# Jadikan Git repository
git init

Initialized empty Git repository in C:/project-website/.git/

git status

Melihat status file: mana yang berubah, mana yang siap di-commit.

Cek StatusGIT
git status

On branch main
Changes not staged for commit:
  modified:   index.php

Untracked files:
  style.css

Untracked = file baru belum pernah di-track Git. Modified = file lama yang diubah.

git add

Memasukkan file ke staging area (siap di-commit).

Menambah File ke StagingGIT
# Tambah 1 file
git add index.php

# Tambah semua file
git add .

# Tambah file tertentu dengan pattern
git add *.php

git commit

Menyimpan perubahan dari staging area ke repository.

Commit dengan PesanGIT 
git commit -m "Menambahkan halaman login"

[main a1b2c3d] Menambahkan halaman login
 2 files changed, 50 insertions(+)

💡 Tips Commit Message

Tulis pesan yang jelas! Bukan "update" atau "fix" doang. Contoh bagus:

  • "Menambahkan validasi form login"
  • "Memperbaiki bug stok minus"
  • "Mengubah warna tombol submit"

git log

Melihat history commit.

Lihat HistoryGIT
git log --oneline

a1b2c3d Menambahkan halaman login
b2c3d4e Setup project awal
c3d4e5f Initial commit

git diff

Melihat perubahan apa saja yang dilakukan pada file.

Lihat PerubahanGIT
git diff index.php

- echo "Hello World";
+ echo "Hello Git!";

- merah = baris dihapus, + hijau = baris ditambah

6️⃣ Alur Kerja Git (Workflow)

Ini alur standar saat kamu ngoding dengan Git:

1 Edit File

Kamu ngoding seperti biasa. Tambah fitur, fix bug, dll.

# Workflow harian

7️⃣ Branching Dasar

Apa itu Branch?

Branch = cabang development terpisah. Kamu bisa kerja di branch terpisah tanpa mengganggu kode utama (main).

🌳 Ilustrasi Branch
      feature-login
         ↙
main ●───●───●───●───●
              ↘
            fix-bug

Kenapa Branch Penting?

  • ✅ Bisa eksperimen tanpa merusak kode utama
  • ✅ Kerja paralel (kamu bikin fitur A, teman bikin fitur B)
  • ✅ Kode utama (main) selalu stabil

Perintah Branch

Perintah BranchGIT
# Lihat semua branch
git branch

# Buat branch baru
git branch fitur-login

# Pindah ke branch
git checkout fitur-login
# atau (Git versi baru)
git switch fitur-login

# Buat dan langsung pindah
git checkout -b fitur-baru

# Gabung branch ke main
git checkout main
git merge fitur-login

GitHub adalah platform cloud untuk menyimpan repository Git. Seperti Google Drive, tapi khusus untuk kode program.

🔧 Git

Software yang berjalan di komputer kamu

Untuk version control lokal

Website untuk hosting repository

Untuk backup & kolaborasi

Butuh internet

Manfaat GitHub untuk Siswa

  • ☁️ Backup - Project aman di cloud
  • 👥 Kolaborasi - Kerja kelompok jadi mudah
  • 📂 Portofolio - Showcase project untuk PKL/kerja
  • 🎓 Belajar - Bisa lihat kode open source

🔟 Menghubungkan Git Lokal ke GitHub

Konsep Remote

Remote = repository Git yang ada di server (GitHub). Biasanya dinamakan origin.

Menghubungkan Repo Lokal ke GitHub

Setup RemoteGIT
# Tambah remote (URL dari GitHub)
git remote add origin https://github.com/username/nama-repo.git

# Cek remote
git remote -v

# Push pertama kali
git push -u origin main

Perintah Remote Penting

Command Fungsi
git push Upload commit ke GitHub
git pull Download & merge dari GitHub
git clone [url] Copy repo GitHub ke lokal
git fetch Download tanpa merge

1️⃣2️⃣ Kolaborasi Tim

Kerja Kelompok dengan GitHub

  1. Satu orang buat repo → invite teman sebagai collaborator
  2. Teman clone repo ke komputer mereka
  3. Masing-masing kerja di branch sendiri
  4. Push ke GitHub, merge ke main

Menghindari Konflik

⚠️ Konflik terjadi kalau:

Dua orang edit baris yang sama di file yang sama!

Cara Menghindari Konflik:

  • ✅ Selalu git pull sebelum mulai kerja
  • ✅ Bagi tugas dengan jelas (kamu edit file A, teman edit file B)
  • ✅ Commit dan push sering-sering, jangan numpuk
  • ✅ Pakai branch untuk setiap fitur

1️⃣7️⃣ Undo & Rollback - Balik ke Versi Sebelumnya

Ini bagian PALING PENTING yang sering ditanyakan murid! Bagaimana kalau salah dan mau balik?

🔄 Kapan Butuh Rollback?

😱 "Waduh, kode error semua setelah saya ubah!"

😰 "Mau balik ke commit terakhir yang masih jalan"

🤦 "Salah tulis commit message, mau ganti"

😅 "Sudah push, tapi mau batalin"

Kasus 1: Belum Add - Buang Perubahan

Kamu edit file, belum git add, mau buang semua perubahan:

Buang perubahan di satu fileGIT
# Buang perubahan di 1 file (kembali ke commit terakhir)
git checkout -- index.php

# Atau Git versi baru
git restore index.php

# Buang SEMUA perubahan
git checkout -- .
git restore .

⚠️ Hati-hati!

Perubahan yang dibuang TIDAK BISA dikembalikan! Pastikan kamu yakin.

Kasus 2: Sudah Add, Belum Commit - Unstage

Sudah git add, mau keluarkan dari staging:

Keluarkan dari stagingGIT 
# Keluarkan 1 file dari staging
git reset HEAD index.php

# Atau Git versi baru
git restore --staged index.php

# Keluarkan semua dari staging
git reset HEAD .

Kasus 3: Sudah Commit, Belum Push - Undo Commit

Sudah commit tapi baru sadar ada kesalahan:

Undo commit terakhirGIT 
# Batalkan commit terakhir, TAPI simpan perubahan file
git reset --soft HEAD~1

# Batalkan commit terakhir DAN buang perubahan file
git reset --hard HEAD~1

# Keterangan:
# HEAD~1 = 1 commit ke belakang
# HEAD~2 = 2 commit ke belakang
# --soft = simpan perubahan
# --hard = buang perubahan

Kasus 4: Mau Lihat/Kembali ke Commit Tertentu

Mau cek kode di commit yang lama:

Checkout ke commit tertentuGIT
# Lihat history dulu
git log --oneline
a1b2c3d Commit terbaru
b2c3d4e Commit sebelumnya
c3d4e5f Commit lama

# Checkout ke commit tertentu (mode lihat saja)
git checkout b2c3d4e

# Mau balik ke yang terbaru
git checkout main

Kasus 5: Sudah Push - Mau Revert

Sudah push ke GitHub, mau batalkan commit (AMAN untuk tim):

Revert commit (buat commit kebalikan)GIT
# Revert = buat commit BARU yang membatalkan commit lama
git revert a1b2c3d

# Git akan buka editor untuk commit message
# Simpan dan tutup

# Push revert ke GitHub
git push

✅ Kenapa pakai revert, bukan reset?

git revert lebih aman karena tidak menghapus history. Cocok kalau sudah push atau kerja tim. git reset menghapus history, bisa bikin konflik kalau sudah push.

Kasus 6: Ganti Commit Message Terakhir

Salah tulis commit message, mau ganti:

Ubah commit message terakhirGIT
# Ubah commit message terakhir (BELUM push)
git commit --amend -m "Pesan baru yang benar"

# Kalau SUDAH push, harus force push (hati-hati!)
git commit --amend -m "Pesan baru yang benar"
git push --force

⚠️ Tentang Force Push

git push --force berbahaya kalau kerja tim! Bisa menimpa kerjaan teman. Pakai hanya kalau kamu kerja sendiri!

Tabel Ringkasan Undo

Situasi Command
Buang perubahan (belum add) git restore namafile
Keluarkan dari staging git restore --staged namafile
Undo commit (simpan file) git reset --soft HEAD~1
Undo commit (buang file) git reset --hard HEAD~1
Batalkan commit yang sudah push git revert <hash>
Ganti commit message git commit --amend -m "..."

1️⃣8️⃣ FAQ - Pertanyaan yang Sering Ditanya Siswa

Ini kumpulan pertanyaan yang pasti akan muncul dari siswa:

❓ "Pak/Bu, saya sudah commit tapi kodenya error semua. Gimana baliknya?"

Jawab: Ada 2 cara:

  1. Kalau belum push: git reset --hard HEAD~1 (kembali 1 commit, buang perubahan)
  2. Kalau sudah push: git revert HEAD (buat commit kebalikan)

❓ "Gimana cara balik ke commit yang kemarin?"

Jawab:

  1. Lihat history: git log --oneline
  2. Copy hash commit yang mau dituju (contoh: b2c3d4e)
  3. Checkout: git checkout b2c3d4e
  4. Kalau mau balik lagi: git checkout main

❓ "Saya sudah push, tapi mau ganti commit message-nya. Bisa?"

Jawab: Bisa, tapi hati-hati:

  1. git commit --amend -m "Pesan baru"
  2. git push --force

⚠️ Warning: Force push bisa berbahaya kalau kerja tim. Pakai hanya kalau kerja sendiri!

❓ "Apa bedanya reset dan revert?"

reset revert
Menghapus commit dari history Membuat commit BARU yang kebalikan
History jadi bersih History tetap lengkap
Bahaya kalau sudah push Aman untuk yang sudah push
Pakai kalau belum push Pakai kalau sudah push / kerja tim

❓ "Saya edit file, tapi belum add. Mau buang semua perubahan. Gimana?"

Jawab: git restore namafile.php atau git restore . untuk semua file.

⚠️ Hati-hati, perubahan yang dibuang tidak bisa dikembalikan!

❓ "Apa itu HEAD?"

Jawab: HEAD adalah pointer ke commit yang sedang aktif (biasanya commit terakhir di branch saat ini).

  • HEAD = commit saat ini
  • HEAD~1 = 1 commit sebelumnya
  • HEAD~2 = 2 commit sebelumnya

❓ "Folder .git itu apa? Boleh dihapus?"

Jawab: Folder .git adalah database Git yang menyimpan semua history, commit, branch, dll.

❌ JANGAN DIHAPUS! Kalau dihapus, semua history Git akan hilang dan folder jadi biasa (bukan repo lagi).

❓ "Gimana caranya hapus file dari Git tapi tetap ada di folder?"

Jawab: Pakai git rm --cached namafile

File akan dihapus dari tracking Git, tapi tetap ada di folder. Jangan lupa tambahkan ke .gitignore kalau tidak mau di-track lagi.

❓ "Kenapa git push ditolak?"

Jawab: Biasanya karena ada commit baru di GitHub yang belum kamu punya. Solusi:

  1. git pull dulu
  2. Kalau ada conflict, selesaikan
  3. Baru git push

❓ "Saya mau mulai dari awal. Gimana hapus semua history Git?"

Jawab: Hapus folder .git, lalu git init lagi:

  1. Hapus folder .git (hidden folder)
  2. git init
  3. git add .
  4. git commit -m "Fresh start"

⚠️ Semua history lama akan hilang!

2️⃣0️⃣ Evaluasi

Soal Pemahaman (10 Soal)

1. Apa fungsi utama Git?

Jawaban: Version control - merekam dan melacak perubahan kode dari waktu ke waktu.

2. Apa perintah untuk inisialisasi Git di folder?

Jawaban: git init

3. Apa perbedaan git add dan git commit?

Jawaban: git add memasukkan file ke staging area. git commit menyimpan perubahan dari staging ke repository.

4. Apa itu staging area?

Jawaban: Tempat penampungan sementara file yang sudah di-add sebelum di-commit.

5. Apa perintah untuk melihat history commit?

Jawaban: git log atau git log --oneline

6. Apa perbedaan Git dan GitHub?

Jawaban: Git = software version control di lokal. GitHub = platform cloud untuk hosting repository Git.

7. Apa perintah untuk upload commit ke GitHub?

Jawaban: git push

8. Apa perintah untuk download update dari GitHub?

Jawaban: git pull

9. Apa itu branch?

Jawaban: Cabang development terpisah yang memungkinkan kerja paralel tanpa mengganggu kode utama.

10. Apa fungsi file .gitignore?

Jawaban: Untuk menentukan file/folder yang tidak perlu di-track oleh Git.

Soal Logika Kasus (5 Soal)

11. Kamu sudah edit 3 file tapi cuma mau commit 1 file saja. Apa yang dilakukan?

Jawaban: Pakai git add namafile.php untuk add 1 file saja, bukan git add .

12. Error "failed to push some refs". Apa yang harus dilakukan?

Jawaban: Jalankan git pull dulu untuk download update terbaru, baru git push

13. Kamu mau lihat perubahan apa saja yang sudah dilakukan sebelum commit. Pakai perintah apa?

Jawaban: git diff untuk lihat perubahan, atau git status untuk lihat file apa saja yang berubah.

14. Teman kamu mau ikut kerja di project GitHub. Langkah apa yang dia lakukan?

Jawaban: git clone https://github.com/username/repo.git untuk copy repository ke komputer dia.

15. Kamu mau bikin fitur baru tanpa mengganggu kode utama. Apa yang dilakukan?

Jawaban: Buat branch baru dengan git checkout -b nama-fitur, kerja di situ, lalu merge ke main kalau sudah selesai.
