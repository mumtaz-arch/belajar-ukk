📦 Analisis Sistem Inventaris Gudang

Dokumen Analisis untuk UKK Level 3 Web Development

1. Latar Belakang

🔴 Masalah yang Dihadapi

Admin gudang sering menghadapi berbagai kendala dalam mengelola inventaris secara manual:

  • Pencatatan Manual: Data barang dicatat di buku atau spreadsheet yang rentan hilang dan rusak
  • Kesalahan Perhitungan: Stok sering tidak akurat karena human error saat menghitung manual
  • Laporan Lambat: Membuat laporan membutuhkan waktu lama karena harus mengumpulkan data dari berbagai sumber
  • Tidak Real-time: Stok tidak terupdate secara otomatis saat ada barang masuk atau keluar
  • Sulit Tracking: Sulit melacak riwayat transaksi barang

🟢 Solusi yang Ditawarkan

Sistem Inventaris Gudang berbasis web yang menyediakan:

  • Digitalisasi Data: Semua data barang tersimpan aman di database
  • Stok Otomatis: Stok terupdate otomatis setiap ada transaksi
  • Laporan Cepat: Export laporan ke PDF dan Excel dalam hitungan detik
  • Validasi Stok: Sistem mencegah stok menjadi minus (tidak valid)
  • Riwayat Lengkap: Semua transaksi tercatat dengan tanggal dan waktu

6. Entity Relationship Diagram (ERD)

Struktur database sistem inventaris gudang:

Tabel: users

Kolom Tipe Data Constraint Keterangan
id INT PRIMARY KEY, AUTO_INCREMENT ID unik user
username VARCHAR(50) UNIQUE, NOT NULL Username untuk login
password VARCHAR(255) NOT NULL Password (bcrypt hash)
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Waktu pembuatan akun

Tabel: barang

Kolom Tipe Data Constraint Keterangan
id INT PRIMARY KEY, AUTO_INCREMENT ID unik barang
nama_barang VARCHAR(100) NOT NULL Nama barang
stok INT DEFAULT 0 Jumlah stok tersedia
harga DECIMAL(12,2) DEFAULT 0 Harga satuan barang
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Waktu data dibuat
updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP Waktu terakhir diupdate

Tabel: transaksi

Kolom Tipe Data Constraint Keterangan
id INT PRIMARY KEY, AUTO_INCREMENT ID unik transaksi
id_barang INT FOREIGN KEY → barang(id) Referensi ke tabel barang
jenis_transaksi ENUM('masuk','keluar') NOT NULL Jenis transaksi
jumlah INT NOT NULL Jumlah barang
tanggal_transaksi DATETIME DEFAULT CURRENT_TIMESTAMP Waktu transaksi
keterangan TEXT - Catatan transaksi

Relasi Antar Tabel

barang (1) ←→ (N) transaksi
Satu barang dapat memiliki banyak transaksi (one-to-many)

Foreign Key: transaksi.id_barang → barang.id
ON DELETE CASCADE: Jika barang dihapus, semua transaksi terkait ikut terhapus

7. Kesimpulan

Berdasarkan analisis yang telah dilakukan, sistem inventaris gudang ini dirancang untuk:

  1. Menyelesaikan masalah pencatatan manual dengan sistem database terstruktur
  2. Mengotomasi perhitungan stok sehingga mengurangi human error
  3. Mempercepat pembuatan laporan dengan fitur export PDF dan Excel
  4. Meningkatkan keamanan data dengan autentikasi dan validasi yang baik
  5. Memberikan visibilitas melalui dashboard dengan statistik dan grafik

Teknologi yang Digunakan

  • Backend: PHP Native, MySQL/MariaDB
  • Frontend: Bootstrap 5, SweetAlert2, DataTables, Chart.js
  • Font: Exo 2 (Google Fonts)
  • Icons: Heroicons

Sistem ini cocok untuk pembelajaran UKK Level 3 karena:

  • Kode sederhana dan mudah dipahami
  • Mencakup operasi CRUD lengkap
  • Logika bisnis yang jelas (stok tidak boleh minus)
  • Fitur export laporan
  • UI modern dan responsif
