Latihan menghadapi pertanyaan dan tugas dari assessor

💡 Tips Menghadapi Assessor:
  1. Jawab dengan percaya diri, jangan ragu-ragu
  2. Jelaskan dengan bahasa sendiri, jangan hafalan
  3. Kalau tidak tahu, bilang "Saya belum tahu, tapi saya akan pelajari"
  4. Tunjukkan lokasi kode di file saat menjelaskan

📝 Sesi 1: Pertanyaan Teori

Assessor akan bertanya tentang konsep dasar. Ini pertanyaan yang sering muncul:

Pertanyaan 1

"Jelaskan apa itu PDO dan kenapa kamu pakai PDO?"

Contoh Jawaban:

"PDO adalah PHP Data Objects, yaitu cara untuk mengakses database dari PHP. Saya pakai PDO karena:

  1. Lebih aman dari SQL Injection dengan prepared statement
  2. Bisa dipakai untuk berbagai database (MySQL, PostgreSQL)
  3. Error handling lebih baik dengan try-catch"

File: app/config/koneksi.php

Pertanyaan 2

"Apa itu session dan bagaimana cara kerjanya di sistem login?"

Contoh Jawaban:

"Session adalah cara menyimpan data user sementara di server selama browsing. Cara kerjanya:

  1. User login dengan username & password
  2. Sistem verifikasi dengan database
  3. Kalau cocok, simpan ID user ke $_SESSION['user_id']
  4. Setiap halaman cek apakah session ada untuk memastikan user sudah login"

File: app/auth/login.php

Pertanyaan 3

"Jelaskan apa itu CRUD!"

Contoh Jawaban:

"CRUD adalah 4 operasi dasar database:

  • Create = INSERT INTO (tambah data)
  • Read = SELECT (baca data)
  • Update = UPDATE (edit data)
  • Delete = DELETE (hapus data)"

File: app/barang/ (index, tambah, edit, hapus)

Pertanyaan 4

"Apa arti dari t.* dan b.nama_barang di query ini?"

SELECT t.*, b.nama_barang FROM transaksi t JOIN barang b ON t.id_barang = b.id

Contoh Jawaban:
  • t adalah alias (nama pendek) untuk tabel transaksi
  • b adalah alias untuk tabel barang
  • t.* artinya ambil SEMUA kolom dari tabel transaksi
  • b.nama_barang artinya ambil HANYA kolom nama_barang dari tabel barang
  • ON t.id_barang = b.id adalah syarat penggabungan

File: app/transaksi/riwayat.php

Pertanyaan 5

"Apa itu SQL Injection dan bagaimana cara mencegahnya?"

Contoh Jawaban:

"SQL Injection adalah serangan dimana hacker memasukkan kode SQL berbahaya melalui input form.

Cara mencegah: Pakai prepared statement dengan placeholder ?

$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$username]);

🔧 Sesi 2: Tugas Praktik

Assessor akan meminta kamu melakukan perubahan pada sistem. Ini simulasinya:

Tugas 1

"Coba tambahkan 1 barang baru dengan nama 'Monitor LG' dan stok 10"

Yang harus dilakukan:

Langkah-langkah:
  1. Buka halaman localhost/bahan-ajar-ukk/app/barang/
  2. Klik tombol "Tambah Barang"
  3. Isi form: Nama = "Monitor LG", Stok = 10, Harga = (isi sesuai)
  4. Klik Simpan
  5. Tunjukkan data sudah masuk di tabel

Jelaskan: "Data disimpan ke database dengan query INSERT INTO di file app/barang/tambah.php"

Tugas 2

"Edit barang yang tadi, ubah stoknya menjadi 15"

Langkah-langkah:
  1. Di tabel barang, klik tombol Edit pada "Monitor LG"
  2. Ubah stok dari 10 menjadi 15
  3. Klik Simpan
  4. Tunjukkan perubahan di tabel

Jelaskan: "Data diupdate dengan query UPDATE SET WHERE di file app/barang/edit.php"

Tugas 3

"Catat transaksi barang masuk: Monitor LG sebanyak 5 unit"

Langkah-langkah:
  1. Buka menu Transaksi → Barang Masuk
  2. Pilih barang "Monitor LG"
  3. Isi jumlah = 5
  4. Klik Simpan
  5. Tunjukkan stok bertambah (15 → 20)

Jelaskan: "Sistem menggunakan transaction database. INSERT ke tabel transaksi dan UPDATE stok barang. Kalau salah satu gagal, semua dibatalkan (rollback)."

Tugas 4

"Tunjukkan dimana kode untuk validasi stok saat barang keluar!"

Langkah-langkah:
  1. Buka file app/transaksi/keluar.php
  2. Cari bagian ini:

if ($barang['stok'] < $jumlah) {
  $error = "Stok tidak cukup!";
}

Jelaskan: "Sebelum mengurangi stok, sistem cek dulu apakah stok cukup. Kalau tidak cukup, tampilkan error dan batalkan transaksi."

Tugas 5

"Export data transaksi ke Excel"

Langkah-langkah:
  1. Buka menu Laporan
  2. Pilih rentang tanggal
  3. Klik tombol Export Excel
  4. File akan terdownload

Jelaskan: "Export menggunakan format CSV yang bisa dibuka di Excel. File ada di app/laporan/export_excel.php"

🔥 Sesi 3: Pertanyaan Lanjutan (Jebakan!)

Pertanyaan ini lebih sulit. Assessor mungkin akan menguji pemahamanmu lebih dalam:

Jebakan 1

"Kenapa kamu proses form di atas, sebelum include header?"

Contoh Jawaban:

"Karena kalau sudah ada output HTML (dari header), fungsi header() untuk redirect tidak bisa dipakai. PHP akan error 'headers already sent'. Jadi harus proses form dulu, baru output HTML."

Jebakan 2

"Apa bedanya fetch() dan fetchAll()?"

Contoh Jawaban:
  • fetch() = Ambil 1 baris hasil (biasanya untuk SELECT WHERE id = ?)
  • fetchAll() = Ambil semua baris hasil (untuk SELECT * atau dengan WHERE yang hasilnya banyak)
Jebakan 3

"Kalau barang dihapus, apa yang terjadi dengan transaksinya?"

Contoh Jawaban:

"Karena pakai ON DELETE CASCADE, transaksi yang terkait dengan barang tersebut akan ikut terhapus otomatis. Ini didefinisikan di FOREIGN KEY di file database.sql."

Jebakan 4

"Kenapa pakai prepared statement, bukan query biasa?"

Contoh Jawaban:

"Untuk mencegah SQL Injection. Prepared statement memisahkan query dari data, jadi data user tidak bisa dieksekusi sebagai SQL. Lebih aman."

Jebakan 5

"Tunjukkan bagian mana yang mencegah XSS!"

Contoh Jawaban:

"Di bagian tampilan HTML, data dari database dibungkus dengan htmlspecialchars(). Contoh:

<?= htmlspecialchars($data['nama']) ?>

Ini mengubah karakter seperti < dan > menjadi entities HTML yang aman."
