🏷️ ALIAS - Nama Pendek untuk Tabel

Sebelum pakai JOIN, kita harus paham dulu tentang ALIAS. Ini penting banget!

💡 Analogi: Nama Panggilan

Nama lengkap: "Muhammad Fakhri Mumtaz Basyari"

Nama panggilan: "Fakhri"

Sama-sama merujuk ke orang yang sama, tapi yang satu lebih pendek!

Step 1: Tanpa Alias (Ribet!)

Query TANPA alias SQL 
SELECT transaksi.id, barang.nama_barang, transaksi.jumlah
FROM transaksi
JOIN barang ON transaksi.id_barang = barang.id;

Lihat? Nama tabel transaksi dan barang ditulis berkali-kali. Bayangkan kalau nama tabelnya panjang seperti detail_transaksi_barang_gudang. Ribet kan?

Step 2: Dengan Alias (Ringkas!)

Query DENGAN alias SQL 
SELECT t.id, b.nama_barang, t.jumlah
FROM transaksi t
JOIN barang b ON t.id_barang = b.id;
1 Cara Membuat Alias

Tulis nama tabel diikuti spasi lalu nama alias:

  • FROM transaksi t → tabel transaksi sekarang bisa dipanggil t
  • JOIN barang b → tabel barang sekarang bisa dipanggil b
2 Cara Pakai Alias

Setelah membuat alias, pakai alias.nama_kolom untuk mengakses kolom:

  • t.id → kolom id dari tabel transaksi
  • t.jumlah → kolom jumlah dari tabel transaksi
  • b.nama_barang → kolom nama_barang dari tabel barang
  • b.id → kolom id dari tabel barang

Step 3: Apa Arti t.* (Bintang)?

* dalam SQL artinya "SEMUA KOLOM".

📊 Perbedaan SELECT
SELECT *

Ambil SEMUA kolom dari SEMUA tabel

SELECT t.*

Ambil SEMUA kolom dari tabel t saja

SELECT t.id

Ambil HANYA kolom id dari tabel t

Contoh Konkret:

Penjelasan t.* dan b.nama_barang SQL 
-- Query yang ada di project kita
SELECT t.*, b.nama_barang
FROM transaksi t
JOIN barang b ON t.id_barang = b.id;

Artinya apa?

t.* = Ambil SEMUA kolom dari tabel transaksi:
  • ✅ t.id
  • ✅ t.id_barang
  • ✅ t.jenis_transaksi
  • ✅ t.jumlah
  • ✅ t.tanggal_transaksi
  • ✅ t.keterangan
b.nama_barang = Ambil HANYA kolom nama_barang dari tabel barang:
  • ❌ b.id (tidak diambil)
  • ✅ b.nama_barang (diambil)
  • ❌ b.stok (tidak diambil)
  • ❌ b.harga (tidak diambil)
Hasil Query = Gabungan keduanya:

id, id_barang, jenis_transaksi, jumlah, tanggal_transaksi, keterangan, nama_barang

🔗 JOIN - Menggabungkan Tabel

Sekarang kita sudah paham ALIAS. Saatnya belajar JOIN!

📌 INNER JOIN (Yang Paling Sering Dipakai)

INNER JOIN = Hanya tampilkan data yang ADA DI KEDUA TABEL.

📊 Visualisasi INNER JOIN

Area BIRU = hasil INNER JOIN (hanya yang cocok)

Step-by-Step: Bagaimana JOIN Bekerja?

1 Database membaca tabel TRANSAKSI

Baris pertama: id_barang = 1

2 Database mencari di tabel BARANG

"Barang dengan id = 1 itu apa ya?" → Ketemu! Laptop ASUS

3 Database menggabungkan hasilnya

Transaksi (id_barang=1) + Barang (id=1) = Hasil lengkap dengan nama barang!

Contoh INNER JOIN SQL 
SELECT t.id, b.nama_barang, t.jenis_transaksi, t.jumlah
FROM transaksi t
JOIN barang b ON t.id_barang = b.id;

-- Penjelasan per baris:
-- FROM transaksi t     = Mulai dari tabel transaksi (alias: t)
-- JOIN barang b        = Gabungkan dengan tabel barang (alias: b)
-- ON t.id_barang = b.id = Syaratnya: id_barang di transaksi SAMA dengan id di barang

Hasil Query:

t.idb.nama_barangt.jenis_transaksit.jumlah
1Laptop ASUSmasuk5
2Mouse Logitechkeluar3
3Laptop ASUSkeluar2

✅ Berhasil!

Sekarang kita bisa lihat nama barang (dari tabel barang) di hasil query, padahal tabel transaksi cuma simpan id_barang!

📌 LEFT JOIN, RIGHT JOIN, FULL JOIN

Selain INNER JOIN, ada jenis JOIN lain:

JenisHasilContoh Kasus
INNER JOIN Hanya data yang cocok di KEDUA tabel Tampilkan transaksi yang barangnya masih ada
LEFT JOIN SEMUA dari tabel kiri + yang cocok dari kanan Tampilkan SEMUA barang, meski belum ada transaksinya
RIGHT JOIN SEMUA dari tabel kanan + yang cocok dari kiri Jarang dipakai (sama saja balik posisi LEFT JOIN)
FULL JOIN SEMUA dari KEDUA tabel MySQL tidak support!

📍 Yang Dipakai di Project

Di project Inventaris ini, kita pakai INNER JOIN (atau cuma ditulis JOIN) karena kita hanya mau tampilkan transaksi yang barangnya masih ada di database.

🔄 CASCADE - Aturan Hapus & Update

CASCADE adalah aturan yang mengontrol apa yang terjadi pada data anak ketika data induk dihapus atau diupdate.

💡 Analogi: Keluarga

  • Tabel BARANG = Orang tua
  • Tabel TRANSAKSI = Anak-anak

Pertanyaan: Kalau orang tua "dihapus" dari database, apa yang terjadi pada anak-anaknya?

Jenis-jenis ON DELETE

🔴 ON DELETE CASCADE
✅ Dipakai di Project

Hapus induk → Anak IKUT TERHAPUS

Contoh: Hapus "Laptop ASUS" → Semua transaksi laptop ikut hilang

🔒 ON DELETE RESTRICT
❌ Tidak Dipakai

Tolak hapus kalau masih ada anak

Contoh: Tidak bisa hapus barang kalau masih ada transaksinya

⚫ ON DELETE SET NULL
❌ Tidak Dipakai

Hapus induk → FK anak jadi NULL

Contoh: Hapus barang → id_barang di transaksi jadi NULL

Contoh ON DELETE CASCADE

🗑️ SEBELUM Hapus Barang id=1

Semua transaksi dengan id_barang=1 OTOMATIS TERHAPUS!

Kode di database.sql

database.sql SQL 
CREATE TABLE transaksi (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_barang INT NOT NULL,
    jenis_transaksi ENUM('masuk', 'keluar'),
    jumlah INT NOT NULL,
    
    -- Foreign Key dengan CASCADE
    FOREIGN KEY (id_barang) REFERENCES barang(id) ON DELETE CASCADE
);

-- Penjelasan:
-- FOREIGN KEY (id_barang)        = Kolom id_barang adalah foreign key
-- REFERENCES barang(id)          = Mengacu ke kolom id di tabel barang
-- ON DELETE CASCADE              = Kalau barang dihapus, transaksi ikut terhapus

⚠️ Kesalahan Umum yang Sering Terjadi

❌ Kesalahan 1: Lupa pakai alias

-- SALAH! Ambiguous column 'id'
SELECT id, nama_barang FROM transaksi 
JOIN barang ON id_barang = id;

-- Error: Column 'id' in field list is ambiguous
-- Karena KEDUA tabel punya kolom 'id'!

Solusi: Pakai alias untuk membedakan kolom dari tabel mana.

-- BENAR!
SELECT t.id, b.nama_barang FROM transaksi t 
JOIN barang b ON t.id_barang = b.id;

❌ Kesalahan 2: Salah kondisi ON

-- SALAH! 
SELECT * FROM transaksi t 
JOIN barang b ON t.id = b.id;
-- Harusnya t.id_barang = b.id, bukan t.id = b.id!

Solusi: Pastikan ON menghubungkan kolom yang benar (foreign key ke primary key).

❌ Kesalahan 3: Lupa bahwa MySQL tidak support FULL JOIN

-- SALAH di MySQL!
SELECT * FROM transaksi t 
FULL JOIN barang b ON t.id_barang = b.id;
-- Error: You have an error in your SQL syntax

Solusi: Pakai INNER JOIN atau LEFT JOIN saja.
